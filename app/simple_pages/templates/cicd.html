GitHub Actions

My project is using GitHub actions to test my changes, upload my docker container to DockerHub, and send my project to my production environment on Heroku. First my changes I made are tested by pytest to verify there wasn't any errors. Then my docker container is created in GitHub Actions, so it can be uploaded directly to DockerHub. Finally, my app will be uploaded to Heroku using the Heroku API where the app will be pushed into a production environment.

Review Code before Merging

I am able to use my local docker container as a development server to review my code. The docker container is instantly updating the changes I make when I save the files I am working on. This allows me to see my changes in real-time to verify my changes worked properly before I merge pull requests.

Deploy Docker Image to DockerHub

  1. I am able to use my local docker container as a development server to review my code. The docker container is instantly updating the changes I make when I save the files I am working on. This allows me to see my changes in real-time to verify my changes worked properly before I merge pull requests.
  2. I am using Heroku as my production server and my code is deployed to Heroku if the test with GitHub Actions is successful. This will only happen if the GitHub action is successful otherwise the test will fail and nothing will be uploaded to the production server.
  3. I am able to deploy my project's docker image to DockerHub by using GitHub Actions. The docker image is created into a container by GitHub actions because GitHub actions is trying to verify that the container has no issues being built. Once the GitHub action is successful it will upload the docker image to DockerHub.
