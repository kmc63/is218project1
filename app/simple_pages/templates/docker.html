Images

A docker is image is a configuration file for docker containers. Docker images are like a list of instructions that are completed in order to build a docker container. This docker image can be used to build as many docker containers as you like. Also, another way to look at docker images is to think of them as a snapshot of a virtual machine.

Containers

A docker container is a lightweight, standalone application that is based off the instructions from a docker image. The docker container will have the bare minimum of configuration and dependencies installed to get its desired application running. The greatest benefit of containers is they take much less hardware to run than VMs which allows you to run much more containers than VMs.

Docker In The Terminal

Docker Commands

  • "docker -version": This command will get the current installation version of docker.
  • "docker pull": This command is used to pull docker images from a dockerhub repository.
  • "docker run": This command is used to run a docker image that you specify as an option.
  • "docker ps": This command will list all running docker containers with their container ID.
  • "docker exec": This command will allow you to enter a running docker container to access it files.
  • "docker kill": This command along with container ID will stop the docker container that has the matching container ID.
  • "docker images": This command will list all docker images stored on your system.
  • "docker rm": This command along withe container ID will delete a stopped docker container.
  • "docker rmi": This command along with image ID will delete the image from your local storage.
