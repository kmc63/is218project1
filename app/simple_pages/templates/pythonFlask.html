Testing with Pytest

Testing your code in general before uploading to Github is good practice and can bring many benefits. The main point is of testing your code is that it gives you confidence that the changes you made are working properly. Writing and maintaining tests can be very difficult but if you use pytest you will have an easier time. I did not code much in python but I was able to use pytest to make sure my homepage and content pages worked properly.

Simple Pages and Testing

I created five simple pages to produce content for my web page. I used the same template for most of my content pages by using rows and columns to evenly divide up the html pages. Then I used headers, paragraphs, images, and unordered lists to fill in the content on my pages. I tested my simple pages by using pytest to check if the page returns a 200 server message. If it returns a 200 server message that means the simple page didn't have any errors.
