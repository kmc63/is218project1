Testing with Pytest

Testing your code in general before uploading to Github is good practice and can bring many benefits. The main point is of testing your code is that it gives you confidence that the changes you made are working properly. Writing and maintaining tests can be very difficult but if you use pytest you will have an easier time. I did not code much in python but I was able to use pytest to make sure my homepage and content pages worked properly.

Simple Pages and Testing

I created five simple pages to produce content for my web page. I used the same template for most of my content pages by using rows and columns to evenly divide up the html pages. Then I used headers, paragraphs, images, and unordered lists to fill in the content on my pages. I tested my simple pages by using pytest to check if the page returns a 200 server message. If it returns a 200 server message that means the simple page didn't have any errors.

Important Files

  • dev.yml: This file will create the environment that will run pytest and then pushes the build to the development heroku app.
  • prod.yml: This file will create the environment that will run pytest, then push the docker file to dockerhub, and then pushes the build to the production heroku app
  • cicd.html: This file is used to create my electronic document to display my continuous integration and continuous development page to the internet.
  • docker.html: This file is used to create my electronic document to display my docker page to the internet.
  • git.html: This file is used to create my electronic document to display my git page to the internet.
  • index.html: This file is used to create my electronic document to display my index page to the internet.
  • pythonFlask.html: This file is used to create my electronic document to display my python/flask page to the internet.
  • style.css: This file is used to apply styles to my web pages.
  • base.html: This file is used as a base html for my web pages by providing a html page to build off that helps constantly used elements.
  • run.py: This file allows for Gunicorn to serve the app in the heroku production environment.
  • simple_pages_test.py: This file is used to test the simple html pages I made. It is testing for a server message 200 to confirm the html is error free.
  • docker-compose.yml: This is an orchestration file that is used to create a docker container that lets you see local development in real-time.
  • Dockerfile: This file is used to create a docker image then the image can create a docker container.
  • heroku.yml: This is a configuration file to get the requirements to use heroku ready.
  • pytest.ini: This is a configuration file to get pytest running properly in the environment.
  • setup.py: This is a python script that is making sure the required packages are installed when the app starts.
