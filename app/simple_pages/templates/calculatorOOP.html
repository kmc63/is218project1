Calculator Program Demonstrating OOP

Encapsulation

Encapsulation is when objects keep their states private and other objects don't have direct access. Other objects must call the public functions of these objects to manipulate its values.

An easy to understand example of encapsulation is to think of a cat as an object. This cat object has its own private values of mood, hungry, and energy with a method meow. Since they're private values no other method can directly change them but instead can use the cat class' public methods to manipulate the object. The cat class has three methods sleep, play, and feed. These methods all have a way to change the original values in the cat object. Which is the main purpose of encapsulation by keeping its value private but allowing the values to be manipulated by its public methods.

The calculator program uses encapsulation in most of its classes to make sure no object can directly access another object. It achieves this by allowing objects to manipulate other objects by using the method of that particular object. For example, an object of the addition class can only be used if the object calling on the addition class uses the add method. When this happens the add method will take the parameters in and calculate the problem then it will return the value of adding two numbers.

Abstraction

Abstraction is when each object only exposes a high level mechanism for using it, this mechanism should only reveal operations relevant to other objects.

An example of abstraction in the real world is the use of your cell phone. You just need to understand what the buttons do on your phone, but you don't need to know the internal details of how the phone processes the button click. As the user of the phone you just need to know that when you press the lock button the phone will lock, and you don't need to know anything else.

The calculator program uses abstraction when the calculations' python file imports the operations class. The calculator does this by calling the class object Add and its method add to calculate a value. the calculation addition object does not care how the operations object gets its value. All the addition object cares about is that the operations class object and method return the correct value.

Inheritance

Inheritance is when you create a child class by deriving from another parent class. The child reuses the fields and methods from the parent. Then this child class can use its own unique fields and methods.

An example of inheritance in simple terms, is trying to look at a person class that has two unique values. Then we have two other class types of teacher and student which both inherit the unique values from the person class. Although, the student and teacher classes inherit the person class they will also have their own unique values and methods. This is the easiest way to understand inheritance from outside a program.

The calculator program is using inheritance when the calculations file imports the operations file's classes into its file. Once the classes are imported the calculations file's own classes will inherit the methods and values from the class objects in the operations file. Once the calculations file inherits the operations file, then the addition class is able to use the inherited clas and have its own unique values and methods.
